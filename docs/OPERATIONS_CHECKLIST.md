## 8. 장애 발생 시 우선 확인 순서

### 빌드 실패

1. 최근 수정 파일 확인
2. Astro route, dynamic route 설정 확인
3. 타입 오류, import 오류 확인
4. 로컬 npm run build 재실행

### 배포 반영 안 됨

1. GitHub 머지 상태 확인
2. Cloudflare Pages 최근 배포 로그 확인
3. 배포 사이트 강력 새로고침
4. 실제 브랜치, 커밋이 맞는지 확인

### CMS 저장 이상

1. Worker OAuth 정상 여부 확인
2. GitHub 권한 상태 확인
3. 저장 후 GitHub 커밋 생성 여부 확인

## 9. 운영 원칙

- 로컬 즉시 반영보다 CMS → GitHub → Cloudflare Pages → 배포 사이트 확인 흐름을 기본으로 삼는다.
- 새 기능 작업 전에는 가능하면 새 작업 브랜치를 먼저 만든다.
- 수정 후에는 항상 코드 확인 → 빌드 → 커밋 → 푸시 → PR → 머지 → 배포 확인 순서를 유지한다.
- 기능이 줄어들거나 삭제되면 반드시 이유를 기록한다.
